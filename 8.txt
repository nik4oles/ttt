 DEV стенд: В основном выполняются unit и integration tests.
 IFT стенд: Проходят function и system tests для проверки функциональности и системной работоспособности.

Scripted Pipeline : Гибкий, использует Groovy код для сложных сценариев.
Declarative Pipeline: Структурированный, проще в написании с четко определенными разделами.

Jenkinsfile - Файл с определением Jenkins Pipeline для автоматизации и версионирования процессов CI/CD.

Smoke-тесты - проверяют основную работоспособность приложения после развертывания.
Это базовая проверка и для полной верификации функциональности требуются дополнительные тесты.
Артефакт попадает в Nexus после успешной сборки.
